Terminal colors

The colors often used are:

for fg in {30..37} {90..97}; do                
    for bg in {40..47}; do                     
        printf "\033[$bg;${fg}m $bg;$fg \033[m"
    done                                       
    echo ""                                    
done

In the beginning, there were no colors, only:

  • bold â€” ESC[1m
  • underline â€” ESC[4m
  • blink â€” ESC[5m
  • inverted â€” ESC[7m

Then the first 8 basic colors were added (black, red, green, yellow, blue, magenta, cyan, white; 30-37m).

Many terminals supported "bold-as-bright", so that there were 16 colors. That is \e[1;34m = \e[94m.

Eventually 256, and finally 24-bit color, were added.

The palette above was printed using:

for i in {0..255}; do                      
    printf "\033[48;5;${i}m %3s \033[m" $i 
    [ $(( (i-15) % 36 )) -ne 0 ] || echo ""
done

Note that the palette colors do not have defined RGB values, so different terminals may display slightly different colors.

As mentioned, that they can also be set to custom values, for example Solarized sets: 

   0    1    2    3    4    5    6    7    8

Remember that these also affect 30-37m.

DOS ANSI.SYS and BBS software implement the same color escape codes as Unix/Linux, so we can have fun reading 1990s DOS art files like:

I wrote an ANSI to HTML script to capture terminal output. Example usage includes: 


vim -c 'redraw!' -c 'set t_te=' -c 'q' file | ,ansi.sh -o html -w70

or cat-ing screenlog file from:


screen                                         
export TERM=screen-256color                    
# C-a i / C-a H to start writing to screenlog.n
...
exit

Some things I learned writing the script:

  • A lot can be achieved without installing anything, just using shell+awk+od(octal dump)
    • But AWK debugging and globally scoped variables is painful for large scripts
    • Working with hex strings is inconvenient
    • It's not fast
    • Having tests helps a lot
  • "The Terminal Escape Sequences Ocean is Deep and Dark"
    • (stolen title which I loved)
    • There are many escape code variations
  • stty can be used with file descriptors other than the terminal's stdin 0.
